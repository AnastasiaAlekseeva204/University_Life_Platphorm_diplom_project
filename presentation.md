## Слайд 2: Проблема, которую решает проект

- **Проблема:** Фрагментированность каналов распространения информации о внеучебных мероприятиях (соцсети, объявления) приводит к потере возможностей для самореализации студентов и отсутствию инструментов контроля активности администрацией.
- **Последствия:** Снижение вовлеченности молодежи, неэффективность управления внеучебной жизнью, отсутствие систематизированного учета достижений.

---

## Слайд 3: Целевая аудитория

- **Основная аудитория:** Студенты МПГУ (бакалавры, магистранты), организаторы мероприятий, администрация университета (деканаты, студсоветы).
- **Дополнительная:** Сотрудники МПГУ, волонтеры, потенциальные партнеры для интеграции с другими вузами.
- **Потребности:** Доступ к актуальной информации, удобная регистрация, рейтинги активности, автоматизированный учет.

---

## Слайд 4: Состояние проблемы

- **Текущее состояние:** Вузы используют Moodle, VK-группы или бумажные объявления, но они не интегрируют парсинг официальных источников, рейтинговые системы и динамические портфолио.
- **Анализ конкурентов:**
  - Moodle: Базовые функции, но без автоматизации парсинга.
  - CampusGroups: Социальные аспекты, но не специализированы на учет достижений.
  - Отечественные решения (МГУ, СПбГУ): Ограничены, без Telegram-ботов и реал-тайм визуализации.

---

## Слайд 5: Новизна проекта

- **Инновации:**
  - Алгоритм динамического портфолио студента (Achievement + StudentEvent) с автоматическим ранжированием по rating_points.
  - Интеграция парсера с визуализацией реал-тайм работы.
  - Поддержка Telegram-ботов для публикации, CKEditor для редактирования.
- **Отличия:** Комбинация автоматизации, персонализации и масштабируемости, не имеющая аналогов в университетских системах.

---

## Слайд 6: Методы проектирования

- **Методы:** Объектно-ориентированное программирование (Python, Django), реляционное моделирование БД (PostgreSQL), парсинг веб-страниц (BeautifulSoup), UX/UI дизайн (Bootstrap).
- **Инструменты:** Django ORM для моделей, requests для HTTP, Pillow для изображений, Git для версионирования.
- **Принципы:** MVC-архитектура, конструктивная согласованность (интеграция компонентов для единой экосистемы).

---

## Слайд 7: Методология проекта

- **Основания:** Теории образовательных технологий (цифровая трансформация вузов), концепции баз данных (реляционные модели), принципы веб-разработки (REST, responsive design).
- **Теории:**
  - Конструктивная согласованность: Все модули (парсер, UI, БД) согласованы для достижения цели.
  - Мотивационные теории (рейтинги для вовлеченности).
- **Концепции:** Персонализация (роли пользователей), автоматизация (парсер), масштабируемость (Django).

---

## Слайд 8: Структура проекта

- **Архитектура:**
  - Backend: Django (myplatform, events_parser).
  - Frontend: HTML/Bootstrap.
  - БД: PostgreSQL.
  - Парсер: BeautifulSoup + requests.
- **Модули:**
  - Модели: User, Event, Community, Faculty, ParsedEvent.
  - Views: Страницы событий, сообществ, профиля.
  - Admin: Управление данными.

## Слайд 9: Примеры фрагментов/модулей

- **Модель User:** Роли (Admin, Student, Organizer), связи с факультетами и событиями.
- **Парсер:** Функция fetch_event_detail() для извлечения данных с сайта МПГУ.
- **UI:** Шаблон event_detail.html с формой регистрации.
- **БД:** Связи многие-ко-многим для участников мероприятий.

## Слайд 10: Цель проекта

- **Цель:** Создание централизованной платформы для повышения осведомленности о мероприятиях, систематизации учета активности и мотивации студентов.
- **Задачи:** Анализ требований, проектирование БД, реализация парсера, разработка UI, тестирование.
